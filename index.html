Modding Knowledge Base
Quick-reference rules, terrain data, and best practices for all tools
▾
Map Scaling Rules
  • Never nearest-neighbor scale raw tile GIDs
  • Upscale terrain-type map, then re-derive transitions
  • Always check all 8 neighbors — not just 4 cardinal
  • Corner tiles require diagonal neighbor detection
  • Transitions must remain exactly 1 tile wide
Layer Handling
  • Ground: terrain-type upscaling + re-derive transitions
  • Items: reposition resources, increase tree density
  • Units: reposition only — never duplicate spawn entities
  • Set: cosmetic, nearest-neighbor or leave empty
  • Resource pools stay exactly 3×3
Placement Rules
  • Never place units, resources or trees on water tiles
  • Never place items on transition / edge tiles
  • Resources are 3×3 — keep clusters intact
  • Nudge spawn points off water after scaling
  • Increase tree density without water overlap
Fill Terrain GIDs
  • DeepWater → 333
  • Ice → 404
  • LongGrass → 429
  • Mountain → 469
  • Sand → 504
  • ShallowWater → 546 (water)
  • Snow → 553
  • Stone → 640
  • Water → 669 (water)
Thumbnail Standards
  • Max size: 160×160 px (engine limit)
  • Output filename = mapname.png (exact match)
  • White dots mark resource pool locations
  • Team numbers in dark circles at spawn positions
  • Ensure labels visible on large maps
Transition Tilesets
  • LongGrass↔ShallowWater: 877–927
  • Snow↔Ice: 1459–1479
  • Mountain↔Sand: 1078–1098
  • Sand↔ShallowWater: 1354–1374
  • Mountain↔LongGrass: 985–1005
  • Snow↔LongGrass: 1645–1665
  • LongGrass↔Stone: 1666–1686
